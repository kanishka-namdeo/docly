# Docs - User Documentation

## Purpose

User-facing documentation for the Doc Assistant application, covering installation, usage, development, and troubleshooting.

## Ownership

This AGENTS.md covers the `docs/` directory:
- `USER_GUIDE.md` — Feature overview and usage instructions
- `INSTALLATION.md` — Installation instructions for all platforms
- `TROUBLESHOOTING.md` — Common issues and solutions
- `ARCHITECTURE.md` — Technical architecture documentation
- `API_REFERENCE.md` — Backend REST API documentation
- `DEVELOPMENT.md` — Developer guide for contributors
- `superpowers/` — Design specs and implementation plans

## Local Contracts

### Documentation Standards
- Markdown (.md) format
- Clear headings and structure
- Code examples where applicable
- Step-by-step instructions
- Prerequisites and assumptions documented

### File Purposes
- **USER_GUIDE.md** — End user guide (getting started, features, usage)
- **INSTALLATION.md** — Installation instructions (system requirements, platform-specific steps)
- **TROUBLESHOOTING.md** — Common issues and fixes (backend, embeddings, LLM, database)
- **ARCHITECTURE.md** — Technical architecture (system design, component diagrams)
- **API_REFERENCE.md** — API documentation (endpoints, schemas, examples)
- **DEVELOPMENT.md** — Developer guide (setup, code structure, testing, contributions)

## Work Guidance

### Writing Documentation
- Use clear, concise language
- Provide step-by-step instructions
- Include prerequisites and assumptions
- Link to related documentation
- Keep examples practical and relevant

### Maintenance
- Update documentation when features change
- Review for accuracy periodically
- Add troubleshooting entries for common issues
- Keep API reference in sync with code changes

## Verification

- Review documentation for accuracy against current codebase
- Verify all links and references are valid
- Test installation and setup instructions
- Validate code examples compile/run correctly
